The Problem

Field personnel can't be authenticated without internet

  • Remote locations have zero or unstable network connectivity — traditional auth systems fail completely
  • Attendance fraud is rampant — photos, printed cards, and screens can fool basic face detection
  • Enterprise AI models are too heavy to ship inside a mobile app (hundreds of MBs)
  • Attendance data recorded offline never reaches the server reliably

Target Environment

Field workers in construction sites, remote factories, and outdoor locations with no stable internet access

Core Challenge

100% offline authentication with anti-spoofing, on a mid-range Android/iOS device, in under 1 second

Anti-Spoofing

5-Step Liveness Engine — no spoofing with photos or screens

1

Look Straight

Yaw ±10°, Pitch ±15°, 20 stable frames required

2

Blink Twice

Detects closed→open transitions. Requires 2 blinks minimum

3

Turn Head Left

Yaw angle < -15° detected via ML Kit landmarks

4

Turn Head Right

Yaw angle > +15° — proves real 3D head movement

5

Smile

Smiling probability > 0.7 — impossible to fake with a still photo

  • All logic runs entirely on-device — zero server calls, zero latency
  • State machine in LivenessEngine.ts — completely decoupled from UI
04 / 10
Edge AI

MobileFaceNet — 2MB model, 192-dim embeddings

  • TFLite quantized model — runs on CPU, no GPU required. Works on Android 8.0+ and iOS 12+
  • 192-dimensional face embeddings — rich feature representation per face
  • Cosine similarity matching — threshold 0.6 balances accuracy vs false positives
  • react-native-fast-tflite v3 — synchronous inference via ArrayBuffer, zero bridge overhead
  • Bundled in app via Metro — no download required at runtime

Inference Pipeline

  • Capture photo on liveness pass
  • Decode image → Float32Array (160×160×3)
  • model.runSync([tensor.buffer])
  • Output: Float32Array (192-dim)
  • Compare vs all enrolled embeddings
  • Return best match above threshold

Model Size vs Accuracy

MobileFaceNet achieves 99.28% on LFW benchmark at just 2MB — designed specifically for mobile face recognition

05 / 10
Architecture

Offline-first with automatic sync & purge

On-Device (Always Works)

  • Camera + face detection (ML Kit)
  • Liveness engine (5-step state machine)
  • MobileFaceNet TFLite inference
  • Employee enrollment + storage
  • SQLite attendance records
  • GPS coordinates per record
  • Live dashboard with stats

Cloud (When Online)

  • NetInfo monitors connectivity
  • syncPending() on app launch
  • syncPending() on reconnect
  • syncPending() after each record
  • POST to API Gateway → Lambda
  • Lambda writes to DynamoDB
  • Local record purged on 200 OK
  • Retry every 2 min if fails

AWS Stack

  • API Gateway (HTTP API)
  • Lambda (Node.js 22)
  • DynamoDB (On-demand)
  • x-api-key authentication
  • syncStatus=1 on write
  • Zero cost at hackathon scale
